# 大黑PPT - 网页演示文稿生成器

将文案和素材生成为单文件 HTML 演示文稿，用于视频录屏。

## 核心铁律

**一页之内的东西，必须讲得非常清楚。** 这是最高优先级原则：
- 每页只有一个明确的重点，一目了然
- 宁可多分页，也不要把内容堆在一页里
- 视觉重心要突出——标题大、重点醒目、留白充足
- 解释性文字和标题要大，图片可以适当缩小（用户后期可放大）
- 简洁是最好的排版

## 工作流程

### 阶段 1: 了解需求

1. 询问用户素材所在的工作目录路径
2. 读取目录下的所有文件，识别：
   - 文案文件（.md、.txt）
   - 图片文件（.jpg、.png、.webp 等）
   - 数据文件（.xlsx、.csv）
3. 分析文案内容，理解演示的结构和逻辑
4. 如果有 Excel/CSV，提取其中的数据和嵌入图片
5. 向用户确认：
   - 演示的整体结构（几页、每页内容）
   - 是否有特殊的布局需求
   - 输出文件的存放位置

### 阶段 2: 规划页面结构

根据文案内容规划每一页：
- 确定总页数和每页的核心内容
- 每页只聚焦一个主题/要点
- 规划每页的逐步揭示顺序（按键依次出现的元素）
- 对于内容过多的页面，果断拆分成多页
- 长文本（如详细提示词）考虑使用左右分栏布局

向用户展示页面规划，确认后再开始编码。

### 阶段 3: 提取素材

如果有 Excel 文件包含嵌入图片：
1. 用 Python + openpyxl + zipfile 提取图片
2. 解析 drawing XML 确定图片与单元格的对应关系
3. 按有意义的名称重命名图片，存放到 images/ 子目录

### 阶段 4: 生成 HTML

生成单个 HTML 文件，遵循以下规范：

**读取设计系统文档：**
```
参考 reference/design_system.md 获取配色、字号、布局规范
```

**技术要求：**
- 单文件 HTML，浏览器直接打开，零依赖
- 3:2 固定比例画面容器（.stage），居中显示，外部黑色背景
- 键盘控制：右箭头/空格 = 下一步，左箭头 = 上一步
- 逐步揭示动画：每按一次键显示一个元素（淡入 + 上移）
- 页面切换：淡入淡出过渡
- 不显示页码和按键提示（录屏用途）
- 进度条保留在底部

**文件命名：**
- 格式：`序号-内容名称.html`，如 `01-软件架构演示.html`、`02-新旧版本对比.html`
- **序号规则**：
  1. 检查工作目录下已有的 `*.html` 文件
  2. 如果没有 HTML 文件，序号为 `01`
  3. 如果已有文件，找出最大序号 +1（如已有 01、02、03，则新文件为 04）
- 根据演示内容起一个有意义的中文文件名
- **禁止使用** `presentation.html`、`demo.html` 等无序号通用名称
- 文件存放在用户指定的工作目录下

**页面类型：**
1. **封面页** - 标题居中，副标题，关键信息标签
2. **标准内容页** - 标题 + 内容区（文字、图片、或两者组合）
3. **对比页** - 标题 + 多张图片并排 + 标签
4. **长文本分栏页** - 标题 + 左侧文本 + 右侧图片
5. **数据表格页** - 标题 + 表格
6. **总结页** - 汇总数据或结论

**布局决策原则：**
- 默认：标题在上，内容在下
- 图片 3 张以内：横向并排
- 图片 + 长文本：左右分栏（文本左，图片右）
- 纯文字要点：大字号居中，逐条揭示
- 有原图对比的：先展示原图（小尺寸），再展示结果图

### 阶段 5: 验证和交付

1. 在浏览器中打开生成的 HTML 文件
2. 告知用户文件位置和操作方式
3. 等待用户反馈，按需调整

## 设计规范速查

详细规范见 `reference/design_system.md`，核心要点：

- **比例**: 3:2（宽:高），`.stage` 容器强制比例
- **配色**: 米白背景 #FAF9F6 + 深黑文字 #1A1A1A + 橙色强调 #FF6B00
- **图文混排页标题**: 1.8-2.4rem，编号 1.1-1.3rem
- **纯文字页标题**: 4.0-4.5rem，正文至少 1.6rem，强调语至少 2.4rem
- **标签**: 1rem+，分数数字 1.8rem+
- **图片**: max-height 限制在 38vh 以内，确保不溢出
- **间距**: 紧凑但不拥挤，padding 28-32px

## 注意事项

- 所有内容必须在 3:2 画面内完整显示，禁止溢出
- 文字宁大勿小——用户在手机小屏上观看
- 图片可以小一些，用户后期会放大处理
- 不要出现滚动条
- 每页元素不要超过 6-7 个（含标题）
- 如果一页放不下，拆成两页
- Excel 提取图片时注意 drawing XML 的行列是 0-indexed

## 实战经验（踩坑记录）

### 字号必须够大
- 纯文字页的字号要比图文页大很多，标题至少 4rem，不要用图文页的 2.4rem 标准
- 用户反馈"字体再大一些"是最常见的修改需求，首次生成就应该用大字号
- 强调语、结论性文字用 2.4rem+，配合橙色背景色块更醒目

### 图片布局：每页 2 张优于 4 张挤一页
- 4 张图用 2×2 grid 会导致每张图太小，观感差
- 推荐拆成两页，每页 2 张并排，图片更大更清晰
- 如果必须用 grid，一定要设 `grid-template-rows: 1fr 1fr` + `min-height: 0`，否则下面的行会溢出不可见

### 连续同主题页面
- 多页展示同一主题时（如截图分两页），后续页的标题不加 `.step`
- 避免同一个标题重复渐入渐出，用户会觉得突兀
- 标题直接随页面出现，只有内容元素做逐步揭示

### 神奇移动 (Magic Move)
- **核心概念**：当元素在页面切换时没有本质变化（如手机、电脑图标），不要让它们"消失再出现"，而是保持元素不变，只改变位置
- **实现方式**：
  1. 元素作为独立层存在，不属于任何 `.page` 容器
  2. 通过 CSS class 切换来改变位置（如 `.bottom` class）
  3. 使用 `transition` 实现平滑移动动画
- **代码结构示例**：
  ```css
  .devices-wrapper {
      position: absolute;
      top: 50%;           /* 第一页位置 */
      left: 50%;
      transform: translate(-50%, -50%);
      transition: top 0.8s ease, transform 0.8s ease;
  }
  .devices-wrapper.bottom {
      top: 88%;           /* 第二页位置 */
      transform: translate(-50%, -100%);
  }
  ```
- **动画顺序**：元素移动 → 显示新元素 → 绘制连接线

### 全屏比例的正确写法
```css
.stage {
    width: 100vw;
    height: calc(100vw * 2 / 3);
    max-height: 100vh;
    max-width: calc(100vh * 3 / 2);
}
```
- 这样写能确保 3:2 比例，同时自动适配屏幕，多余区域显示黑色背景

### 连接线绘制
- 连接线使用 SVG `<path>` + `stroke-dasharray` 动画
- **位置计算**：需要精确计算起点（设备顶部）和终点（服务器底部）坐标
- **弧线效果**：使用贝塞尔曲线 `Q` 命令，控制点放在中间位置
- **动画**：`stroke-dashoffset` 从线长变为 0，实现"生长"效果

### 逐个缩放效果（行内元素依次渐入）

适用于：表格行、卡片列表、图标网格等需要逐个出现的场景。

**核心思路**：父容器控制整行的显示/隐藏，子元素通过 `transition-delay` 实现从左到右依次渐入。

**CSS 结构**：
```css
/* 父行：整行淡入 + 上移 */
.table-row {
    opacity: 0;
    transform: translateY(10px);
    transition: opacity 0.4s ease, transform 0.4s ease;
}
.table-row.visible {
    opacity: 1;
    transform: translateY(0);
}

/* 子卡片：从左滑入，由父级 .visible 触发 */
.app-card {
    opacity: 0;
    transform: translateX(-20px);
    transition: opacity 0.45s ease, transform 0.45s ease;
}
.table-row.visible .app-card {
    opacity: 1;
    transform: translateX(0);
}
```

**HTML 用法**：每个子元素通过内联 `transition-delay` 控制出场顺序：
```html
<div class="table-row" data-row="1">
    <div class="app-card" style="transition-delay: 0s;">第1个</div>
    <div class="app-card" style="transition-delay: 0.12s;">第2个</div>
    <div class="app-card" style="transition-delay: 0.24s;">第3个</div>
</div>
```

**要点**：
- 间隔 0.12s 左右体感最佳，太快看不出波浪，太慢拖沓
- 父行用 `translateY` 上移，子元素用 `translateX` 左滑，两个方向叠加效果更丰富
- 回退时父级移除 `.visible`，子元素自动回到隐藏状态，无需额外处理

### 简笔画漫画风格（四宫格漫画）

适用于：用漫画形式讲故事、解释概念、展示场景。

**画风特点：**
- 黑白简笔画，线条为主，不上色
- 使用 SVG 绘制，保持矢量清晰
- 火柴人风格的人物，简洁但有表现力
- 背景使用标准米白色 `#FAF9F6`

**构图原则：**
- 每页一个场景/一个情节，像漫画格子
- 人物大小根据重要性调整：主角大（占画面 50%+），配角/物品小
- 对话框统一放在**画面中上方**（避免与视频字幕重叠）
- 多人场景：主角居中，配角围绕成圈

**人物绘制技巧：**
```
头部：圆形 + 简单五官（圆点眼睛、弧线嘴巴）
身体：线条（stroke-width 4-5）
表情：通过眉毛、嘴巴变化表达
  - 思考：托腮动作 + 皱眉
  - 踌躇：挠头 + 汗滴
  - 激动：睁大眼睛 + 张嘴
手臂：通过位置表达动作（托腮、张开、拿手机等）
```

**漫画元素：**
- **对话框**：圆角矩形 + 尖角指向说话者，白色填充 + 黑色描边
- **思考框**：同对话框，但可省略小气泡（更简洁）
- **爆炸框**：星形/多边形，黑色填充 + 白色文字，表示突然想到/强调
- **汗滴**：椭圆，表示紧张/踌躇

**SVG 样式模板：**
```css
.stroke-main {
    stroke: #1A1A1A;
    stroke-width: 3;
    fill: none;
    stroke-linecap: round;
    stroke-linejoin: round;
}
.fill-main { fill: #1A1A1A; }
.speech-bubble { fill: #fff; stroke: #1A1A1A; stroke-width: 3; }
.explosion-bubble { fill: #1A1A1A; }
.speech-text { font-size: 24px; font-weight: 700; fill: #1A1A1A; }
```

**文字规范：**
- 对话框文字：24px+，加粗，确保录屏可读
- 爆炸框文字：18px+，白色，加粗

**常见场景模板：**
1. 单人+思考：人物居左占大比例，对话框在右上方
2. 物品特写：物品放大居中，对话框在上方
3. 人+物品互动：人物一侧，物品另一侧，对话框上方
4. 多人围观：主角居中张开双臂，多人围绕成圈
